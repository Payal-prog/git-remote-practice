# Information about this file
## Learning how to use Pull Requests
Learning three kinds of pull requests -
1. Merge pull request
2. Squash and Merge
3. Rebase and merge

## Squash and Merge
A "Squash and Merge" is a GitHub pull request merge strategy that combines all individual commits from a feature branch into a single, consolidated commit before applying them to the default branch  

## Rebase and merge
In GitHub, a Rebase and Merge is a pull request merging strategy that takes all individual commits from your feature branch, rewrites them onto the absolute tip of the base branch, and fast-forwards the base branch. It incorporates all your granular commits without creating a cluttered "merge commit"
