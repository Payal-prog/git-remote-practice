# Rebase and merge
In GitHub, a Rebase and Merge is a pull request merging strategy that takes all individual commits from your feature branch, rewrites them onto the absolute tip of the base branch, and fast-forwards the base branch. It incorporates all your granular commits without creating a cluttered "merge commit"
